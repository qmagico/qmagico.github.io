Seções

  • ContentSection

    • Seção de conteúdos na biblioteca QMágico. Uma seção suporta somente outras seções ou conteúdos.

    Campo Descrição
    name Nome da seção de conteúdos.
    id Identificador único da seção.
    parent_id Identificador da seção pai. Caso o valor seja nulo, a seção solicitada é uma seção raiz
    subsections Número de subseções.
    contents Número de conteúdos.

    Exemplo: 
    
        {'name': 'section',
         'id': 3,
         'parent_id': 1,
         'subsections': 0,
         'contents': 15}

Conteúdos

  • Video

    • Vídeos da plataforma, hospedados no Vimeo e no Youtube.

    Campo Descrição
    id Idenfiticador único de conteúdos.
    type Categoria do vídeo. Existem dois tipo: VIDEO (para vídeos do Youtube) e VIMEO (para vídeos do Vimeo).
    title Título do vídeo na plataforma QMágico (independente do título no Youtube ou no Vimeo).
    likes Número de opções "curtir" que o vídeo recebeu.
    points Número de pontos que se ganha ao assistir o vídeo.
    section_id Identificador da seção que armazena este conteúdo.
    seconds Tempo de duração do vídeo.

    Exemplo: 
     
      
        {'id': 10,
        'type': 'VIDEO',
        'title': 'Matemática - Soma e Subtração',
        'likes': 20,
        'points': 200,
        'section_id': 3,
        'seconds': 80}

  • TextClass

    • Aulas escritas pelos professores.

    Campo Descrição
    id Idenfiticador único de conteúdos.
    type Tipo do conteúdo.
    title Título do vídeo na plataforma QMágico (independente do título no Youtube ou no Vimeo).
    likes Número de opções "curtir" que a aula-texto recebeu.
    points Número de pontos que se ganha ao completar a leitura.
    section_id Identificador da seção que armazena este conteúdo.

    Exemplo: 
     
      
        {'id': 10,
        'type': 'TEXT_CLASS',
        'title': 'Grandes civilizações - Grécia',
        'likes': 8,
        'points': 300,
        'section_id': 3

  • ExerciseList

    • Listas de exercícios para praticar.

    Campo Descrição
    id Idenfiticador único de conteúdos.
    type Tipo do conteúdo.
    title Título da lista de exercícios.
    likes Número de opções "curtir" que a lista de exercícios recebeu.
    points Número de pontos que se ganha ao finalizar a lista de exercícios.
    section_id Identificador da seção que armazena este conteúdo.

    Exemplo: 
     
      
        {'id': 10,
        'type': 'EXERCISE_LIST',
        'title': 'Prática 1 - Por que, Por quê, Porque ou Porquê?',
        'likes': 12,
        'points': 1000,
        'section_id': 3 [or null],
        'content_size': 10}

Métricas

  • InteractionSummary

    • Resumo das interações entre alunos e conteúdos na plataforma.

    Campo Descrição
    status Define se o aluno concluiu ou não o conteudo. É populado com CORRECT quando o conteúdo foi finalizado e INCORRECT quando ainda não foi finalizado.
    type Tipo do conteúdo.
    like Define se o aluno "curtiu" o conteúdo. É populado com as opções LIKE ou DISLIKE.
    spent Tempo (em segundos) que o aluno visualizou o conteúdo.
    acesses Número de acessos que o aluno fez ao conteúdo.
    points Número de pontos que o aluno conseguiu no conteúdo.

    Alguns tipos de conteúdos tem campos adicionais nos seus InteractionSummary.


    Listas de exercícios
    Campo Descrição
    efficiency Número real entre 0 e 1 que armazena a proficiência do aluno na lista de Exercícios.
    correct_answers Contador de exercícios respondidos corretamente.
    mistakes Contador de exercícios respondidos incorretamente.
    solved Contador de exercícios respondidos. É a soma dos campos correct_answers e mistakes.
    tips Contador de dicas pedidas pelo aluno na lista de exercícios.

    Video
    Campo Descrição
    stop Tempo de parada(em segundos)do aluno no vídeo.

    Exemplo: 
    
        {"status": "CORRECT",
         "like": "DISLIKE",
         "spent": 15,
         "accesses": 4,
         "efficiency": 0.725,
         "points": 200,
         "mistakes": 1, 
         "correct_answers": 2, 
         "tips": 0,
         "solved": 3}

Escola

  • Organization

    • Organização de ensino, é a entidade máxima da plafatorma e é única por namespace.

    Campo Descrição
    name Nome da organização de ensino.
    licenses Número de licenças totais da organização de ensino.
    used_licenses Número de licenças usadas pela organização de ensino.
    visitors_count Número de usuários visitantes na plataforma.
    students_count Número de estudantes na plataforma.
    teachers_count Número de professores na plataforma.
    admins_count Número de administradores na plataforma.

    Exemplo: 
    
        {"name": "Sistema brasileiro de ensino - SBE",
         "licenses": 1000,
         "used_licenses": 680,
         "visitors_count": 680,
         "students_count": 570,
         "teachers_count": 100,
         "admins_count": 10}

  • School

    • Escola onde os alunos estudam e os professores ensinam.

    Campo Descrição
    name Nome da escola.
    students_count Número de estudantes na plataforma.
    teachers_count Número de professores na plataforma.

    Exemplo: 
    
        {"name": "Escola Brasil",
         "students_count": 10,
         "teachers_count": 3}

  • SchoolClass

    • Turmas das escolas, agrupam alunos e conteúdos.

    Campo Descrição
    name Nome da turma.
    teacher_id Identificador único do professor/proprietário da turma.
    status Status da turma. Pode ser populado com ACTIVE ou INACTIVE.

    Exemplo: 
    
        {"name": "Turma Química - Módulo 1",
         "teacher_id": "flavio_andrade",
         "teacher_name": "Flávio Andrade",
         "status": "ACTIVE"}

Usuários


  • User

    • Usuários.

    Campo Descrição
    id Idenfiticador único do usuário na plataforma.
    name Nome do usuário.
    roles Lista de permissões do usuário na plataforma. As opções são: 'VISITOR' para visitante, 'STUDENT' para estudante, 'TEACHER' para professor e 'ADMIN' para administrador.
    gender Sexo do usuário, representado por M ou F.
    state Estado informado pelo usuário.
    town Cidade informada pelo usuário.
    school_name Nome da escola do usuário.
    points Pontos ganhos pelo usuário.
    birth Data de nascimento do usuário no formato dd/mm/aaaa.

    Exemplo: 
    
        {"id": giovane.liberato
         "name": "Giovane Liberato",
         "roles": ["ADMIN", "VISITOR", "TEACHER", "STUDENT"],
         "gender": "M",
         "town": "São Paulo",
         "state": "SP",
         "school_name": "FATEC",
         "points": 1800,
         "birth": "10/03/1985"}
